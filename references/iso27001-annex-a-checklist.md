# ISO/IEC 27001:2022 Annex A Checklist

Use with skill `iso27001-annex-a-controls`.

## Organizational controls (A.5)

- [ ] Information security policies defined and approved
- [ ] Roles and responsibilities documented
- [ ] Segregation of duties for ePHI/CHD systems
- [ ] Management responsibilities for ISMS
- [ ] Contact with authorities and special interest groups
- [ ] Threat intelligence process
- [ ] Project management security integrated
- [ ] Asset inventory maintained
- [ ] Acceptable use policies
- [ ] Return of assets on termination

## People controls (A.6)

- [ ] Background verification where appropriate
- [ ] Employment terms include confidentiality
- [ ] Security awareness training (annual minimum)
- [ ] Disciplinary process for violations

## Physical controls (A.7)

- [ ] Physical security perimeters
- [ ] Physical entry controls and logs
- [ ] Securing offices, rooms, and facilities
- [ ] Protecting against environmental threats

## Technological controls (A.8)

- [ ] User endpoint devices managed
- [ ] Privileged access restricted and logged
- [ ] Access rights reviewed periodically
- [ ] Source code access controlled
- [ ] Authentication mechanisms (MFA for admin)
- [ ] Capacity management
- [ ] Malware protection
- [ ] Vulnerability management
- [ ] Configuration management
- [ ] Information deletion when no longer required
- [ ] Data masking / tokenization where applicable
- [ ] Data leakage prevention (Presidio for agents)
- [ ] Backup and restore tested
- [ ] Logging and monitoring (SIEM)
- [ ] Clock synchronization
- [ ] Privileged utility program use
- [ ] Software installation controlled
- [ ] Network security and segregation
- [ ] Web filters / secure browsing for admin
- [ ] Cryptography policy and key management
- [ ] Secure development lifecycle
- [ ] Security testing in development
- [ ] Outsourced development governed
- [ ] Separation of environments
- [ ] Change management
- [ ] Test information protected (no prod PHI)
- [ ] Audit testing planned to minimize impact

## Statement of Applicability

- [ ] Every Annex A control marked: Applicable / Not Applicable with justification
- [ ] Implementation status and evidence owner per applicable control
- [ ] Risk treatment plan linked to gaps
