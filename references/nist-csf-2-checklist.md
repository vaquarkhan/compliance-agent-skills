# NIST Cybersecurity Framework 2.0 Checklist

Use with skill `nist-csf-2-assessment`. Reference: NIST CSF 2.0 (NIST CSWP 29).

## Govern (GV)

- [ ] Organizational context and risk management strategy documented
- [ ] Roles, responsibilities, and authority for cybersecurity defined
- [ ] Policy established and communicated
- [ ] Oversight of cybersecurity risk (board/executive)
- [ ] Supply chain risk management program
- [ ] AI/agent systems included in governance scope

## Identify (ID)

- [ ] Asset inventory (hardware, software, data, agents, MCP servers)
- [ ] Business environment and critical services mapped
- [ ] Risk assessment process with documented criteria
- [ ] Improvement opportunities tracked

## Protect (PR)

- [ ] Identity management and access control (IAM, MFA)
- [ ] Awareness and training
- [ ] Data security (classification, encryption, DLP/PHI redaction)
- [ ] Platform security (hardening, patching)
- [ ] Technology infrastructure resilience

## Detect (DE)

- [ ] Continuous monitoring and anomaly detection
- [ ] Adverse event analysis (SIEM, agent audit logs)
- [ ] MCP tool invocation monitoring

## Respond (RS)

- [ ] Incident response plan tested
- [ ] Communications during incidents
- [ ] Analysis and mitigation procedures
- [ ] Improvements after incidents

## Recover (RC)

- [ ] Recovery plans exercised
- [ ] Restoration of services verified
- [ ] Post-incident improvements incorporated

## Implementation Tiers

Document current tier (Partial → Adaptive) per Function and target tier with gap plan.

## CSF Profile

- [ ] Current Profile documented
- [ ] Target Profile documented
- [ ] Gap analysis exported with prioritized roadmap
